Introducción al Circuito

Este proyecto demuestra el control de LEDs conectados en paralelo mediante Arduino. La configuración en paralelo permite que todos los LEDs reciban el mismo voltaje de manera independiente, siendo controlados simultáneamente por el microcontrolador a través de señales digitales.

El circuito utiliza salidas digitales PWM del Arduino para generar un patrón de parpadeo sincronizado en tres LEDs verdes. Esta práctica ilustra conceptos fundamentales de electrónica digital, control de actuadores y programación de microcontroladores.

Materiales Utilizados

Componentes necesarios para construir el circuito de parpadeo de LEDs en paralelo:

🔵
Arduino UNO
Placa de desarrollo con microcontrolador ATmega328P. Genera las señales digitales de control (HIGH/LOW) para encender y apagar los LEDs mediante programación.
💚
3x LEDs Verdes
Diodos emisores de luz de 5mm con longitud de onda de ~525nm (verde). Voltaje de operación típico: 2.0-2.2V. Corriente máxima: 20mA.
⚡
3x Resistencias 220Ω
Resistencias limitadoras de corriente (código de colores: rojo-rojo-café). Protegen los LEDs del exceso de corriente calculadas mediante la Ley de Ohm.
🔲
Protoboard
Tablero de conexiones sin soldadura con filas interconectadas. Permite el montaje temporal del circuito facilitando pruebas y modificaciones.
🔗
Cables Jumper
Cables de conexión macho-macho para establecer las conexiones entre Arduino y protoboard. Colores: amarillo (señal), azul (señal), rosa (señal), negro (GND).
🔌
Cable USB
Cable USB tipo A a tipo B para programación del Arduino y suministro de alimentación de 5V desde el computador.

Proceso de Construcción Paso a Paso

Guía detallada para ensamblar el circuito de parpadeo de LEDs en paralelo con Arduino:

1

Configuración de Conexiones a Pines Digitales

Conectar los cables de señal desde los pines digitales PWM del Arduino (identificados con el símbolo ~) hacia el protoboard. En esta imagen se observan tres cables de colores (amarillo, azul y rosa) conectados desde los pines digitales del Arduino.

Cada cable lleva la señal de control digital (HIGH = 5V, LOW = 0V) que determinará el encendido y apagado de cada LED. Los pines PWM permiten también controlar la intensidad mediante modulación de ancho de pulso si fuera necesario.

2

Conexión a Tierra (GND)

Establecer la conexión de tierra común (GND) desde el Arduino hacia el protoboard. Este cable (generalmente negro o café) completa el circuito eléctrico y sirve como referencia de voltaje (0V) para todo el sistema.

En la imagen se aprecia el cable conectado desde el pin GND del Arduino. La tierra común es esencial en la configuración paralela, ya que todos los LEDs comparten este punto de retorno de corriente. Sin esta conexión, el circuito no funcionaría correctamente.

3

Montaje de LEDs y Resistencias en Protoboard

Ensamblar los componentes en el protoboard siguiendo la configuración en paralelo. Cada LED verde debe conectarse con su respectiva resistencia de 220Ω en serie para limitar la corriente.

Configuración por LED:
• Ánodo (terminal larga del LED) → Resistencia 220Ω → Cable de señal del Arduino
• Cátodo (terminal corta del LED) → Línea de tierra común (GND)

La imagen muestra los tres LEDs verdes alineados horizontalmente con sus resistencias azules conectadas. Cada LED funciona independientemente pero recibe la misma señal de control, creando un efecto de parpadeo sincronizado.

¿Por Qué Funciona el Circuito?

El funcionamiento de este circuito se basa en varios principios fundamentales de electrónica digital y control de microcontroladores. A continuación se explican los conceptos clave:

⚡ Principios de Funcionamiento

  • Configuración en Paralelo: Los tres LEDs están conectados en paralelo, lo que significa que cada uno tiene su propia rama independiente desde la fuente de voltaje hasta tierra. Esto permite que todos reciban el mismo voltaje (5V cuando el pin está en HIGH) y que la falla de un LED no afecte a los demás. Cada rama es independiente y tiene su propia resistencia limitadora.
  • Control Digital del Arduino: El microcontrolador ATmega328P del Arduino genera señales digitales en sus pines de salida mediante el código programado. Cuando se ejecuta digitalWrite(pin, HIGH), el pin produce 5V; con digitalWrite(pin, LOW), produce 0V. Esta alternancia controlada crea el efecto de parpadeo.
  • Resistencias Limitadoras de Corriente: Cada LED tiene una resistencia de 220Ω en serie calculada mediante la Ley de Ohm: R = (Vfuente - VLED) / ILED = (5V - 2.0V) / 0.02A ≈ 150Ω (se usa 220Ω como valor comercial estándar). Estas resistencias protegen los LEDs limitando la corriente a aproximadamente 13.6mA, por debajo del máximo de 20mA.
  • Polarización del LED: Los LEDs son diodos que solo conducen en una dirección. El ánodo (terminal larga, positiva) se conecta hacia la fuente de voltaje (pin del Arduino), y el cátodo (terminal corta, negativa) hacia tierra. Esta polarización correcta permite el flujo de corriente y la emisión de luz cuando el pin está en estado HIGH.
  • Función delay() en Arduino: La biblioteca de Arduino proporciona la función delay(ms) que pausa la ejecución del programa durante un tiempo especificado en milisegundos. Combinando digitalWrite() con delay(), se crea un patrón de encendido-espera-apagado-espera que produce el efecto de parpadeo visible.
  • Capacidad de Corriente de los Pines: Cada pin digital del Arduino puede proporcionar hasta 40mA de corriente máxima (20mA recomendado). Como cada LED consume ~13.6mA, es posible controlar múltiples LEDs desde diferentes pines sin exceder la capacidad total del microcontrolador (200mA máximo para todos los pines combinados).

🔬 Cálculos Técnicos del Circuito

  • Voltaje de Operación del LED: VLED = 2.0V - 2.2V (típico para LEDs verdes)
  • Corriente por LED: ILED = (VPIN - VLED) / R = (5V - 2.0V) / 220Ω ≈ 13.6mA
  • Potencia Disipada por Resistencia: P = I² × R = (0.0136A)² × 220Ω ≈ 40.7mW
  • Potencia Consumida por LED: PLED = VLED × ILED = 2.0V × 0.0136A ≈ 27.2mW
  • Corriente Total del Circuito: ITOTAL = 3 × ILED = 3 × 13.6mA ≈ 40.8mA
  • Consumo Total de Potencia: PTOTAL ≈ 3 × (40.7mW + 27.2mW) ≈ 203.7mW

💻 Estructura del Código Arduino

El código típico para este circuito sigue esta estructura básica:

  • void setup(): Configuración inicial que se ejecuta una sola vez. Aquí se declaran los pines como salidas usando pinMode(pin, OUTPUT) para los tres LEDs.
  • void loop(): Bucle principal que se repite infinitamente. Contiene la secuencia: digitalWrite(pin, HIGH) para encender → delay(tiempo) para esperar → digitalWrite(pin, LOW) para apagar → delay(tiempo) para esperar antes de repetir.
  • Control Sincronizado: Al usar el mismo delay() para todos los LEDs, se logra que parpadeen al unísono. Para patrones más complejos, se pueden usar diferentes tiempos o técnicas como millis() para control no bloqueante.
